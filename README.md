# CC98 Smart Search

一个面向 CC98 的轻量浏览器搜索增强插件。当前小版本采用“原生重排 + API 别名补召回”：让 CC98 先渲染自己的搜索结果，插件移动原有结果卡片；对于 `线性代数` / `线代` 这类简称，再用 CC98 API 补充召回结果并插入同一个列表，不再生成单独的结果面板。

当前 MVP 支持：

- 多关键词搜索
- 空格分隔的多关键词会分别 API 补搜并合并，例如 `线性代数 韩刚` 会补搜 `线性代数` 和 `韩刚`
- 轻量中文分词，支持 `线性代数期中韩刚` 这类无空格输入
- `+关键词` 必须包含
- `-关键词` 排除
- 基于标题、作者、版面、时间、浏览量的 SearchRank
- API 别名补召回，例如搜 `线性代数` 时补搜 `线代`
- 单字中文短词扩召回，例如搜 `梅` 时补搜 `梅花`、`梅雨`；未知单字会先搜原字，再从 API 返回标题里动态挖候选短语
- 独立别名词库，覆盖 213 组全学科常见课程/学科简称、英文名和老师简称种子
- 综合、时间优先、热度优先、自定义排序
- 点击插件图标开关增强、调节模糊程度、补召回间隔；仅在自定义排序时显示权重设置

模糊程度会影响别名补召回组数和相关性评分。补召回默认 1.2 秒后发起，遇到限流会自动退避重试。API 失败时会保留 CC98 原生结果，只跳过补召回。手工课程词库在 `extension/lexicon.js`，老师首字母词库在 `extension/teacher-lexicon.js`；别名组内词会被当作强等效词处理，例如 `数字逻辑 / 数逻 / digital logic`、`微积分 / vif / 微甲 / 微乙`、`韩刚 / hg`。当前手工课程词库覆盖数学、物理化学、生医、计算机、电信、工科、经管、人文社科、地环农建等常见基础课和专业基础课。

当输入里有空格分隔的多个普通关键词时，插件会保留 CC98 原生组合搜索，同时把每个关键词单独走 API 补召回，去重后插入同一个列表，再统一按 SearchRank 排序。`+关键词` 仍然作为必须包含条件，`-关键词` 不参与补搜并继续过滤结果。

单字中文查询会进入短词扩召回模式：插件先使用 `shortQueryExpansions` 中的手工候选，再从当前标题里挖包含该字的 2-4 字短语。若词库没有覆盖，例如只搜 `桥`，插件会先通过 API 搜原字，并从返回标题中动态挖出包含该字的候选短语继续补搜。排序时会对同一短语簇重复结果施加轻微降权，避免 `杨梅` 这类结果完全淹没 `梅花`、`梅雨` 等其他意图。

排序逻辑：

- 综合：相关性 70% + 时间 15% + 热度 15%
- 时间优先：相关性 40% + 时间 45% + 热度 15%
- 热度优先：相关性 40% + 时间 10% + 热度 50%
- 自定义：使用弹窗中的三个权重滑杆，并按总和归一化

## 安装验证

1. 打开 Chrome：`chrome://extensions/`
2. 开启“开发者模式”
3. 点击“加载已解压的扩展程序”
4. 选择本项目的 `extension` 目录
5. 打开 `https://www.cc98.org/` 并保持登录
6. 在顶部搜索框输入 `线性代数 期末` 或 `+线代 -答案 期末`
7. 按 Enter 或点击搜索图标

进入 CC98 原生搜索页后，结果列表上方出现 `已按 ... SearchRank 原生重排` 提示即为加载成功。提示中会显示 `API 补召回` 数量；鼠标悬停提示条可以看到本次分词，悬停结果卡片可以看到相关性、时间、热度分数。

插件只增强 CC98 的主题/全站/版内搜索。用户搜索和版面搜索暂时交给 CC98 原站处理。

## 词库维护

词库不是穷举分词表，只维护“强等效”和少量手工种子。普通泛词不可能全部提前收录，因此交给 API 返回结果驱动的短语发现处理。课程简称、英文名和手工老师简称维护在 `extension/lexicon.js`：

- `courseAliasGroups`：课程名、简称、英文名，目前按学科块维护 213 组，例如 `["数字逻辑", "数逻", "digital logic"]`
- `teacherAliasGroups`：老师姓名和常用缩写，例如 `["韩刚", "hg"]`
- `shortQueryExpansions`：单字查询的探索候选，例如 `"梅": ["梅花", "梅雨", "青梅"]`
- `segmentTerms`：只用于切分无空格输入的普通词，例如 `期中`、`期末`、`教材`

批量老师首字母词库维护在 `extension/teacher-lexicon.js`。当前从 `https://chalaoshi.de/t/{id}/` 扫描 `1..12000` 生成，得到 `10295` 个有效详情页、`9787` 个去重姓名、`9786` 个姓名/首字母强等效组，最大有效 ID 为 `10933`。

后续新增老师时，不需要改排序算法，只要在 `teacherAliasGroups` 里追加一组即可。
